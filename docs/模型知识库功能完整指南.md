## 功能概述

### 核心功能

在首页新增了"模型知识库"模块，用户可以查询20多种强化学习模型的详细信息。

### 访问位置

```
首页 → 配置区域 → 模型知识库（问题类型选择区域下方）
```

### 提供信息

每个模型的详情页面包含：

| 信息类型 | 说明 |
|---------|------|
| **核心定位** | 模型的核心思想和定位 |
| **技术架构** | 详细的架构说明 |
| **关键创新点** | 突出的创新特性 |
| **性能表现** | 实验数据和性能指标 |
| **优势与局限** | 全面的评价 |
| **适用场景** | 推荐使用场景 |
| **论文链接** | 可直接跳转到原始论文 |

## 使用方法

### 快速开始

#### 1. 选择模型
在"选择模型"下拉菜单中选择您感兴趣的模型。

模型按类别分组：
- **构造方法（自回归）**: AM, POMO, MatNet等
- **构造方法（非自回归）**: DIMES, DIFUSCO等
- **改进方法**: POMO+, Sym-NCO等
- **强化学习算法**: REINFORCE, A2C, PPO等
- **传导式强化学习**: DeepACO, L2D等

#### 2. 查看详情
点击 **🔍 查看详情** 按钮，在新窗口中打开模型详情页面。

#### 3. 浏览信息
```
模型详情页面展示：
├── 基本信息
│   ├── 模型名称
│   ├── 发表年份
│   └── 论文链接
├── 核心定位
│   └── 模型的核心思想
├── 技术架构
│   └── 详细的架构说明
├── 关键创新点
│   └── 突出的创新特性
├── 性能表现
│   └── 实验数据和性能指标
├── 优势与局限
│   └── 全面的评价
└── 适用场景
    └── 推荐使用场景
```

#### 4. 返回首页
点击页面底部的 **← 返回首页** 按钮。

## 模型分类

### 构造方法（自回归 / Autoregressive）

#### AM (Attention Model)
- **定位**: 经典的注意力机制模型
- **特点**: 最易于理解，实现简单
- **性能**: TSP-50 Gap: 1.41%
- **速度**: <1s
- **论文**: Attention, Learn to Solve Routing Problems! (2019)

#### POMO (Policy Optimization with Multiple Optima)
- **定位**: 多起点策略优化
- **特点**: 性能优异，速度快
- **性能**: TSP-50 Gap: 0.89%
- **速度**: <1s
- **论文**: POMO: Policy Optimization with Multiple Optima (2020)
- **推荐**: ⭐ 生产部署首选

#### MatNet (Matrix Encoding Network)
- **定位**: 矩阵编码网络
- **特点**: 表达能力强，大规模问题优势明显
- **性能**: TSP-50 Gap: 0.58%
- **速度**: <1s
- **论文**: MatNet (2021)

#### Sym-NCO (Symmetry-aware Neural Combinatorial Optimization)
- **定位**: 对称感知神经组合优化
- **特点**: 当前SOTA之一，质量最优
- **性能**: TSP-50 Gap: 0.47%
- **速度**: <1s
- **论文**: Sym-NCO (2022)
- **推荐**: ⭐ 学术研究首选

#### MDAM (Multi-Decoder Attention Model)
- **定位**: 多解码器注意力模型
- **特点**: 并行解码，多样性强
- **性能**: TSP-50 Gap: 0.98%
- **速度**: <1s

#### PolyNet
- **定位**: 多项式网络
- **特点**: 高阶交互建模
- **性能**: TSP-50 Gap: 0.68%
- **速度**: <1s

### 构造方法（非自回归 / Non-Autoregressive）

#### DIMES
- **定位**: 扩散模型
- **特点**: 并行生成，速度快
- **状态**: 敬请期待

#### DIFUSCO
- **定位**: 扩散模型用于组合优化
- **特点**: 新兴方法
- **状态**: 敬请期待

### 改进方法

#### POMO+
- **定位**: POMO增强版
- **特点**: 多起点+数据增强
- **性能**: TSP-50 Gap: 0.45%（with augmentation）
- **速度**: ~2m

#### Active Search / EAS
- **定位**: 测试时优化
- **特点**: 质量提升显著
- **性能**: 可在测试时进一步优化
- **论文**: Efficient Active Search (2021)

#### SGBS (Stochastic Beam Search)
- **定位**: 随机束搜索
- **特点**: 搜索与采样结合
- **性能**: 质量与多样性平衡

#### MTPOMO (Multi-Task POMO)
- **定位**: 多任务学习
- **特点**: 知识迁移，一个模型解决多个问题
- **状态**: 敬请期待

### 强化学习算法

#### REINFORCE
- **定位**: 最基础的策略梯度算法
- **特点**: 最简单，易于实现
- **优点**: 原理清晰，适合学习
- **缺点**: 方差大，训练不稳定

#### A2C (Advantage Actor-Critic)
- **定位**: 优势Actor-Critic算法
- **特点**: 更稳定，方差更小
- **优点**: 收敛快，性能好
- **缺点**: 实现稍复杂

#### PPO (Proximal Policy Optimization)
- **定位**: 近端策略优化
- **特点**: 工业界首选，最可靠
- **优点**: 稳定性强，超参数不敏感
- **缺点**: 计算开销稍大
- **推荐**: ⭐ 实际应用首选

### 传导式强化学习

#### DeepACO
- **定位**: 深度蚁群优化
- **特点**: 结合传统优化与深度学习
- **性能**: 大规模问题优势明显
- **论文**: DeepACO (2023)

#### L2D (Learning to Delegate)
- **定位**: 学习委派
- **特点**: 混合优化策略
- **状态**: 敬请期待

## 推荐模型

### 新手入门

#### AM (Attention Model)
- **推荐理由**: 最经典的模型，易于理解
- **适合**: 初学者学习注意力机制
- **优点**: 代码简洁，概念清晰
- **TSP-50 Gap**: 1.41%

#### PtrNet (Pointer Network)
- **推荐理由**: 开创性工作，概念简洁
- **适合**: 理解序列到序列模型
- **优点**: 奠基性论文，值得深入学习

#### REINFORCE
- **推荐理由**: 最基础的强化学习算法
- **适合**: 理解策略梯度
- **优点**: 数学原理清晰

### 追求性能

#### POMO (推荐⭐)
- **推荐理由**: 性能与速度平衡最好
- **适合**: 生产环境部署
- **优点**: 快速(<1s) + 高质量(Gap 0.89%)
- **TSP-50 Gap**: 0.89%

#### Sym-NCO (推荐⭐⭐)
- **推荐理由**: 当前SOTA之一，质量最优
- **适合**: 学术研究
- **优点**: 理论扎实，性能最佳
- **TSP-50 Gap**: 0.47%

#### MatNet
- **推荐理由**: 表达能力强，大规模问题优势明显
- **适合**: 大规模问题（>100节点）
- **优点**: 矩阵编码，扩展性好
- **TSP-50 Gap**: 0.58%

### 创新研究

#### DeepACO
- **推荐理由**: 结合传统优化与深度学习
- **适合**: 探索混合方法
- **优点**: 大规模问题表现优异

#### Active Search / EAS
- **推荐理由**: 测试时优化，质量提升显著
- **适合**: 对质量要求极高的场景
- **优点**: 可在推理时进一步优化

#### MTPOMO
- **推荐理由**: 多任务学习，知识迁移
- **适合**: 需要解决多个相关问题
- **优点**: 一个模型，多个问题

### 训练算法选择

#### REINFORCE
- **适合**: 快速原型，简单问题
- **优点**: 实现最简单
- **缺点**: 方差大

#### A2C
- **适合**: 中等规模问题
- **优点**: 更稳定，方差更小
- **缺点**: 实现稍复杂

#### PPO (推荐⭐)
- **适合**: 生产环境，复杂问题
- **优点**: 工业界首选，最可靠
- **缺点**: 计算开销稍大

## 使用技巧

### 1. 对比学习
```
技巧: 同时打开多个模型页面
目的: 对比它们的优劣势
示例: 对比 AM vs POMO vs Sym-NCO
```

### 2. 由浅入深
```
学习路径:
第1步: 经典模型（AM, PtrNet）
第2步: 改进方法（POMO）
第3步: 现代方法（Sym-NCO, MatNet）
第4步: 创新方法（DeepACO, EAS）
```

### 3. 论文阅读
```
建议:
1. 对感兴趣的模型，点击论文链接深入学习
2. 先读经典论文（AM, POMO）
3. 再读最新论文（Sym-NCO）
```

### 4. 实践验证
```
方法:
1. 在训练页面选择不同模型进行实验
2. 对比实际性能
3. 验证模型描述的准确性
```
