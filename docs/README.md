## 💡 关键概念

### 部署方案

| 方案 | 说明 |
|------|------|
| **本地开发** | 在你的电脑上运行，只有你能访问 |
| **Systemd 服务** | 在 Linux 服务器上运行，使用系统服务管理 |
| **Docker** | 应用打包成容器，便于部署和迁移 |
| **Docker Compose** | 多个容器的组合，包含应用、数据库、Nginx |

### 部署到哪里

| 地方 | 说明 |
|------|------|
| **本地电脑** | 开发、测试，仅供自己使用 |
| **自己的服务器** | 如果你已经有 Linux 服务器 |
| **云平台** | 租赁云服务器（阿里云、腾讯云、AWS 等） |

### 域名和访问

| 概念 | 说明 |
|------|------|
| **局域网** | 只能在同一网络访问（如家里的 WiFi） |
| **互联网** | 全球任何地方都能访问 |
| **域名** | 如 example.com，易于记忆 |
| **HTTPS** | 安全的加密连接，推荐用于生产 |
