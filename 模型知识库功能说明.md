# 模型知识库功能说明

## 功能概述

在网页首页的问题类型选择区域下方，新增了一个"模型知识库"模块，用户可以：
1. 通过下拉菜单选择想要了解的强化学习模型
2. 点击"查看详情"按钮
3. 在新窗口中查看该模型的详细信息

## 实现位置

### 前端部分 (`templates/index.html`)

**添加位置**：问题类型选择区域下方，策略配置区域右侧

**代码结构**：
```html
<div class="config-card">
    <h3>📚 模型知识库</h3>
    <select id="model-info-select">
        <!-- 按类别分组的模型选项 -->
    </select>
    <button onclick="viewModelInfo()">🔍 查看详情</button>
</div>
```

**JavaScript函数**：
```javascript
function viewModelInfo() {
    const modelValue = document.getElementById('model-info-select').value;
    window.open('/model/' + modelValue, '_blank');
}
```

### 后端部分 (`app.py`)

**模型数据库** (`MODEL_DATABASE`)：
包含所有模型的详细信息，每个模型包含：
- `name`: 模型简称
- `full_name`: 完整名称
- `category`: 所属类别
- `year`: 发表年份
- `paper_url`: 论文链接
- `paper_venue`: 发表会议/期刊
- `core_concept`: 核心定位（HTML格式）
- `architecture`: 技术架构与创新（HTML格式）
- `innovations`: 关键创新点（可选）
- `performance`: 性能表现（HTML格式）
- `advantages`: 优势（HTML格式）
- `limitations`: 局限性（HTML格式）
- `applications`: 适用场景（可选）

**路由处理**：
```python
@app.route('/model/<model_id>')
def model_info(model_id):
    if model_id not in MODEL_DATABASE:
        return "模型不存在", 404
    model_data = MODEL_DATABASE[model_id]
    return render_template('model_info.html', model_data=model_data)
```

### 模型详情页面 (`templates/model_info.html`)

**页面结构**：
1. **页面头部**：显示模型名称、分类、年份、论文链接
2. **核心定位**：模型的核心思想和定位
3. **技术架构与创新**：详细的架构说明
4. **关键创新点**：突出的创新特性（如果有）
5. **性能表现**：实验数据和性能指标
6. **优势与局限**：双栏对比显示
7. **适用场景**：推荐使用场景（如果有）
8. **返回按钮**：返回首页

**设计特点**：
- 现代化的渐变紫色主题
- 清晰的卡片式布局
- 响应式设计，适配移动端
- 优雅的悬停效果和过渡动画

## 使用流程

1. **选择模型**：
   - 用户在首页"模型知识库"区域的下拉菜单中选择感兴趣的模型
   - 模型按类别分组显示，方便查找

2. **查看详情**：
   - 点击"🔍 查看详情"按钮
   - 在新标签页中打开模型详情页面

3. **阅读信息**：
   - 浏览模型的核心概念、技术架构、性能数据等
   - 如果提供了论文链接，可以点击查看原始论文

4. **返回首页**：
   - 点击页面底部的"← 返回首页"按钮
   - 或直接关闭标签页

## 技术亮点

### 1. 结构化数据存储
所有模型信息以Python字典形式存储在后端，便于维护和扩展。

### 2. 动态页面渲染
使用Flask的Jinja2模板引擎动态生成页面，支持HTML格式的富文本内容。

### 3. 美观的UI设计
- 渐变色背景
- 卡片式布局
- 清晰的信息层次
- 现代化的交互效果

### 4. 良好的用户体验
- 新窗口打开，不影响当前操作
- 响应式设计，支持各种屏幕尺寸
- 清晰的视觉层次和信息组织

## 扩展说明

### 添加新模型

在 `app.py` 的 `MODEL_DATABASE` 中添加新条目：

```python
"NewModel": {
    "name": "NewModel",
    "full_name": "New Model Name",
    "category": "模型类别",
    "year": "2024",
    "paper_url": "https://...",
    "paper_venue": "Conference/Journal",
    "core_concept": """<p>核心概念...</p>""",
    "architecture": """<p>架构说明...</p>""",
    "performance": """<p>性能数据...</p>""",
    "advantages": """<ul><li>优势1</li></ul>""",
    "limitations": """<ul><li>局限1</li></ul>"""
}
```

然后在 `templates/index.html` 的下拉菜单中添加对应选项：

```html
<option value="NewModel">NewModel - New Model Name</option>
```

## 注意事项

1. 所有模型内容均支持HTML格式，可以包含列表、粗体、链接等
2. 论文链接若不存在可设为 `"#"`
3. `innovations` 和 `applications` 字段是可选的
4. 确保模型ID在字典和HTML中保持一致

## 功能效果

用户现在可以：
- 快速了解各种强化学习模型的核心思想
- 对比不同模型的优劣势
- 通过论文链接深入学习
- 选择最适合自己需求的模型

这为用户提供了一个完整的学习路径，从模型概览到详细信息再到原始论文，形成了良好的知识传递闭环。
